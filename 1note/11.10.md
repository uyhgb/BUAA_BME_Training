# setupbash要设定好RMW通讯才能在终端之间通信
🔑 关键点
每个终端都需要独立 source 环境：

source /home/weeq/unitree_ros2/setup_local_fixed.sh
之所以需要在每个终端 source，是因为这个脚本设置了关键的环境变量：

ROS_LOCALHOST_ONLY=1 - 启用本地发现
ROS_DOMAIN_ID=0 - DDS 域ID
RMW_IMPLEMENTATION=rmw_cyclonedds_cpp - 使用 CycloneDDS
只要所有终端都使用相同的配置（相同的 ROS_DOMAIN_ID、相同的 RMW_IMPLEMENTATION），它们就能互相发现话题。


# 启动流程cheatsheet
```sh
# 每个终端都要source这个local.sh
# 本地回环模式，并且包含了example内部其他setup.bash
cd ~/unitree_ros2
source setup_local_fixed.sh
```
## 启动模拟器
```sh
ros2 run unitree_ros2_example simulate_robot
```

## 查看话题
```sh
ros2 topic list
# 直接监听话题发布
ros2 topic echo /lf/lowstate
# 查看话题信息（可查看订阅者）
ros2 topic info /lf/lowstate
```
